# STALAB 火灾检测系统（含管理后台/主程序）GPU 服务器部署指南（Ubuntu 22.04.5 + Tesla V100）

## 0. 结论：这台服务器能用 GPU 吗？
能用。你这台服务器是 **Tesla V100 16G**，且已安装 NVIDIA Driver / CUDA（你提供的版本信息表明 GPU 运行环境齐全）。只要再确保 **Docker 能调用 GPU（NVIDIA Container Toolkit）**，主程序容器就可以用 `cuda:0` 跑 YOLO 推理。

---

## 1. 部署目标与组件说明
**目标：一台服务器同时运行：**
- **主程序（火灾检测）**：拉取同网段摄像头 RTSP 流，YOLO 推理，报警，写业务日志到文件系统。
- **管理后端（admin-backend）**：FastAPI，管理飞书/短信联系人、系统参数、日志查询接口。
- **管理前端（admin-frontend）**：Vue3 + Nginx 静态站点，提供配置管理、日志查询 UI。

**重要目录（都要持久化到宿主机文件系统）：**
- `admin-backend/config/`：配置 JSON（`feishu.json`、`sms.json`、`system.json`…）
- `log/`：日志文件（按天 `fire_detection_YYYY-MM-DD.log`，默认保留 7 天自动清理）
- `output/`：报警截图（可选持久化）

## 3. 让 Docker 支持 GPU（关键步骤）
要让容器里能使用 GPU，需要安装 **NVIDIA Container Toolkit** 并使用 **CDI（Container Device Interface）** 模式。

### 3.1 安装/启用（Ubuntu 22.04 常规流程）
如果你们服务器已经能在容器里看到 GPU，可跳过这一节；否则按以下做：

```bash
sudo apt-get update
sudo apt-get install -y nvidia-container-toolkit
```

### 3.2 生成 CDI 规范（关键）
```bash
sudo nvidia-ctk cdi generate --output=/etc/cdi/nvidia.yaml
sudo systemctl restart docker
```

### 3.3 验证容器内 GPU
```bash
docker run --rm --device nvidia.com/gpu=all nvidia/cuda:12.4.1-runtime-ubuntu22.04 nvidia-smi
```

能输出 GPU 信息即说明 OK。

### 5.1 admin-backend 的 .env（敏感信息）
编辑 `/opt/fire-detection/admin-backend/.env`，写入飞书/阿里云等密钥（如不使用可留空）：


> 强烈建议：不要使用弱口令/默认口令；也不要把密码写进日志或提交到仓库。

### 5.2 RTSP 地址与系统参数（通过后台页面配置）
部署完成后，在 **管理后台 → 系统参数** 设置：
- `rtsp_url`：摄像头 RTSP 地址（按你给的 TP-LINK 文档方式配置）
- `yolo_device`：建议设为 `cuda:0`（使用 GPU）
- `yolo_confidence`、`consecutive_threshold`、`alert_cooldown_seconds` 等按需调优

**RTSP 地址建议先验证：**
- 在服务器上安装 VLC 或用 `ffmpeg` 先确认 RTSP 可拉流
- 摄像头和服务器需同网段（你服务器 IP 192.168.31.32），确保摄像头 `554` 端口可达

## 6. Docker Compose（生产推荐：三服务一键启动，GPU 加速）
在 `/opt/fire-detection/docker-compose.yml` 写入如下（端口可按需改）：

```yaml
version: "3.9"

services:
  admin-backend:
    build: ./admin-backend
    container_name: fire-admin-backend
    working_dir: /app
    environment:
      - PYTHONUNBUFFERED=1
    ports:
      - "8001:8001"
    volumes:
      - ./admin-backend/config:/app/config
      - ./log:/app/log
      - ./admin-backend/.env:/app/.env:ro
    command: >
      sh -c "python -m uvicorn app.main:app --host 0.0.0.0 --port 8001 --workers 2"
    restart: unless-stopped

  admin-frontend:
    build:
      context: ./admin-frontend
      dockerfile: Dockerfile
    container_name: fire-admin-frontend
    ports:
      - "8080:80"
    depends_on:
      - admin-backend
    restart: unless-stopped

  main-service:
    build: .
    container_name: fire-main
    working_dir: /app
    environment:
      - PYTHONUNBUFFERED=1
    volumes:
      - ./admin-backend/config:/app/admin-backend/config
      - ./log:/app/log
      - ./output:/app/output
      - ./core/yolo/weights:/app/core/yolo/weights
    devices:
      - "nvidia.com/gpu=all"
    command: >
      sh -c "python main.py"
    depends_on:
      - admin-backend
    restart: unless-stopped
```

> 说明：主程序会从 `admin-backend/config/system.json` 读取 `rtsp_url/yolo_device/...` 等参数（你们现在的热加载机制已支持）。

## 8. 重要运行注意：主程序当前会弹窗显示画面
你们的 `main.py` 里使用了 `cv2.imshow(...)`，在纯 headless 环境会报错或卡住。

你们服务器有 TurboVNC，可以两种选择：
1) **用 TurboVNC 登录后运行（有图形环境）**：适合调试阶段  
2) **生产建议改为无界面模式**：禁用 `imshow/waitKey`，只输出日志/报警截图（更稳定）

如果你希望我把主程序改成“服务器默认无界面运行，必要时可开关显示”，我可以直接给出一版改造（带开关参数）。

  - 固定 IP、固定码流
  - 主码流用于检测时，尽量降低到“满足识别”的分辨率/帧率以提升稳定性

## 11. 常见故障排查
- **拉不到 RTSP**
  - 先用 VLC/ffmpeg 在服务器上验证 RTSP 地址
  - 确认摄像头与服务器同网段、554 端口可达、账号密码正确
- **GPU 没用上**
  - `docker run --rm --gpus all nvidia/cuda... nvidia-smi` 先确保 Docker GPU 可用
  - 系统参数里 `yolo_device` 设为 `cuda:0`
- **主程序卡住/报错**
  - 多数是 `cv2.imshow` 在无 GUI 环境导致；按第 8 节处理

### 12.1 增强版火焰检测器
为解决鼠标红灯等误检测问题，新增**火焰特征验证层**：

- **亮度验证**：平均亮度≥80
- **动态验证**：检测静止光源（如鼠标灯）
