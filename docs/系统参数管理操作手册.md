# 系统参数管理操作手册

## 1. 概述

系统参数管理模块提供了火灾检测系统运行参数的可视化配置和管理功能，包括 YOLO 检测参数、报警逻辑参数、硬件配置等。

## 2. 参数分类

### 2.1 YOLO 检测参数
- `yolo_weights`: YOLO 模型权重文件路径（受限参数）
- `yolo_confidence`: YOLO 检测置信度阈值（范围: 0.1-1.0，默认: 0.5）
- `yolo_iou_threshold`: YOLO NMS IOU 阈值（范围: 0.1-1.0，默认: 0.45）
- `yolo_imgsz`: YOLO 输入图像尺寸（可选值: 416, 640, 832, 1024）
- `min_box_area`: 最小检测框面积（范围: 100-50000，默认: 500）
- `max_box_area`: 最大检测框面积（范围: 10000-5000000，默认: 500000）

### 2.2 报警逻辑参数
- `detection_interval`: 检测间隔帧数（范围: 1-30，默认: 5）
- `consecutive_threshold`: 连续检测触发报警次数（范围: 1-50，默认: 5）
- `alert_cooldown_seconds`: 报警冷却时间（范围: 30-3600，默认: 180）
- `confirm_wait_seconds`: 确认等待时间（范围: 30-600，默认: 180）

### 2.3 硬件配置参数
- `yolo_device`: YOLO 推理设备（可选值: cpu, cuda, cuda:0, cuda:1）
- `camera_index`: 摄像头索引（范围: 0-10，默认: 0）
- `rtsp_url`: RTSP 流地址（为空时使用摄像头）

## 3. 参数权限级别

- **只读 (read_only)**: 无法修改的系统参数
- **可编辑 (editable)**: 普通用户可修改的参数
- **受限 (restricted)**: 需要重启才能生效的高级参数

## 4. 前端界面使用指南

### 4.1 访问参数管理页面

1. 启动前端和后端服务
2. 访问前端管理界面
3. 点击左侧菜单的"系统参数"进入参数管理页面

### 4.2 查看和筛选参数

1. **按分类筛选**: 使用下拉菜单选择参数分类（YOLO 检测/报警逻辑/硬件配置/通用配置）
2. **搜索参数**: 在搜索框中输入关键词搜索参数名或描述
3. **刷新列表**: 点击"刷新"按钮重新加载参数列表

### 4.3 编辑参数

1. 在参数列表中找到要编辑的参数
2. 点击右侧的"编辑"按钮（仅可编辑参数可用）
3. 在弹出的编辑对话框中:
   - 查看参数基本信息（参数名、描述）
   - 修改参数值（根据参数类型使用不同的输入组件）
   - 填写修改原因（可选）
   - 填写操作人（可选）
4. 点击"保存"按钮提交修改

### 4.4 查看历史版本

1. 在参数列表中找到要查看的参数
2. 点击右侧的"历史"按钮
3. 在弹出的版本列表中查看该参数的所有历史变更记录
4. 如需回滚，点击对应版本的"回滚"按钮

### 4.5 查看审计日志

1. 点击"审计日志"标签页
2. 查看所有参数操作记录（包括操作类型、参数、变更、操作人、IP地址、时间）

### 4.6 导出/导入配置

**导出配置**:
1. 点击"导出配置"按钮
2. 浏览器将自动下载 `system-config-xxx.json` 配置文件

**导入配置**:
1. 点击"导入配置"按钮
2. 选择之前导出的配置文件
3. 确认导入操作
4. 配置将覆盖当前系统参数

### 4.7 热加载配置

1. 点击"热加载"按钮
2. 系统将重新加载配置文件
3. 适用于手动修改配置文件后的快速刷新

## 5. 后端集成

### 5.1 Python 代码集成

```python
from core.communication.system_config import get_system_config

# 获取系统配置实例
config = get_system_config()

# 获取单个参数
confidence = config.get("yolo_confidence", 0.5)

# 使用便捷方法
threshold = config.get_consecutive_threshold()
cooldown = config.get_alert_cooldown_seconds()

# 注册参数变更回调
def on_config_changed(changed_params):
    print("参数变更:", changed_params)

config.register_change_callback(on_config_changed)
```

### 5.2 主程序集成示例

在 `main.py` 中使用系统参数:

```python
from core.communication.system_config import get_system_config
from core.yolo.detector import Detector

config = get_system_config()

# 使用系统参数初始化检测器
detector = Detector(
    weights_path=config.get("yolo_weights"),
    conf=config.get_yolo_confidence(),
    device=config.get_yolo_device(),
    imgsz=config.get("yolo_imgsz")
)

# 监听参数变更
def on_detector_params_changed(changed):
    if "yolo_confidence" in changed:
        detector.conf = changed["yolo_confidence"]["new"]

config.register_change_callback(on_detector_params_changed)
```

## 6. API 文档

### 6.1 基础信息

- **Base URL**: `http://localhost:8001/api/v1/system`
- **响应格式**: JSON

### 6.2 API 列表

#### 获取参数列表
```
GET /params?category=xxx
```

**查询参数**:
- `category` (可选): 参数分类 (yolo_detection, alarm_logic, hardware, general)

**响应示例**:
```json
{
  "code": 200,
  "msg": "success",
  "data": {
    "params": [...],
    "total_count": 15,
    "last_updated": "2026-03-11T12:00:00"
  }
}
```

#### 获取单个参数
```
GET /params/{key}
```

#### 更新参数
```
PUT /params
Content-Type: application/json

    "yolo_confidence": 0.75,
    "consecutive_threshold": 10

  "change_reason": "优化误报率",

```

#### 获取参数历史版本
```
GET /params/{key}/versions?limit=20
```

#### 回滚参数
```

```

#### 获取审计日志
```
GET /audit-logs?limit=100
```

#### 导出配置
```
GET /export
```

#### 导入配置
```
POST /import
Content-Type: multipart/form-data

operator: admin
```

#### 热加载配置
```
POST /reload
```

#### 获取需要重启的参数
```
GET /restart-required
```

## 7. 注意事项

1. **受限参数**: 修改受限参数后需要重启系统才能生效
2. **参数验证**: 系统会自动验证参数类型和范围，无效值将被拒绝
3. **版本控制**: 所有参数变更都会被记录，可随时回滚
4. **审计日志**: 所有操作都会被记录，包括操作人 IP 地址
5. **热更新**: 大部分参数支持热更新，无需重启即可生效
6. **备份**: 建议在修改参数前先导出配置进行备份

## 8. 故障排查

### Q: 参数修改后没有生效？
A: 检查该参数是否标记为"需重启"，如果是，请重启系统。

### Q: 如何恢复到之前的配置？
A: 可以通过参数的"历史"按钮回滚到指定版本，或使用导入功能恢复备份。

### Q: 前端界面无法连接后端？
A: 请确保后端服务已启动，检查 `VITE_API_BASE_URL` 配置是否正确。

### Q: 配置文件损坏怎么办？
A: 删除 `admin-backend/config/system.json` 文件，系统会自动重新生成默认配置。

## 9. 火灾高级参数（新增）说明（fire 专用，smoke 关闭）

### 9.1 这些参数在代码里有默认值吗
有。当前系统对这批参数采用“代码默认值 + 配置覆盖 + 范围钳制”的机制。

1. 代码默认值来源：`core/yolo/detector.py` 的 `_init_runtime_defaults()`。
2. 热配置覆盖来源：`admin-backend/config/system.json` 的 `params.<key>.value`。
3. 非法值保护：`_refresh_runtime_config()` 会按最小/最大范围做钳制，避免异常值导致逻辑失稳。

当 `system.json` 缺失某个 key 时，会回退到代码默认值，不会崩溃。

### 9.2 参数总览（用途 / 默认值 / 建议填写）

| key | 用途 | 代码默认值 | 建议填写范围（办公室场景） |
|---|---|---:|---|
| `fire_small_area_threshold` | 小目标面积阈值（像素），`area < 该值` 按更严格规则校验 | `1500` | `1500 ~ 2500` |
| `fire_medium_area_threshold` | 中目标面积阈值（像素），用于 small/medium/large 分段 | `6000` | `6000 ~ 10000` |
| `fire_small_target_motion_min` | 小目标最小运动比率，过低视为静止误报风险 | `0.08` | `0.08 ~ 0.15` |
| `fire_small_target_fire_ratio_min` | 小目标最小火焰颜色占比 | `0.10` | `0.10 ~ 0.16` |
| `fire_track_match_dist_px` | 跨帧跟踪关联最大中心距离（像素） | `80` | `50 ~ 80` |
| `fire_track_min_iou` | 跨帧关联最小 IoU | `0.10` | `0.10 ~ 0.25` |
| `fire_track_area_change_max` | 跨帧允许最大面积变化倍率 | `2.0` | `1.4 ~ 2.0` |
| `yolo_iou_threshold` | YOLO NMS IoU 阈值（已接入 `predict(iou=...)`） | `0.45` | `0.40 ~ 0.50` |
| `min_box_area` | fire 校验前的最小框面积过滤（像素） | `200` | `500 ~ 1200` |
| `max_box_area` | fire 校验前的最大框面积过滤（像素） | `500000` | 一般保持默认 |

> 说明：UI 或 `system.json` 中展示的“默认值”可能与代码默认值不同（例如 `min_box_area` 在某些配置模板中为 `500`）。运行时最终生效值以“热配置 value > 代码默认值”优先。



写法 A（左上右下）：
```json
"[{\"x1\":0.27,\"y1\":0.66,\"x2\":0.33,\"y2\":0.76}]"
```

写法 B（左上 + 宽高）：
```json
"[{\"x\":0.27,\"y\":0.66,\"w\":0.06,\"h\":0.10}]"
```

### 9.4 调参方向（快速版）

1. 黄色小物体误报多：优先增大 `min_box_area`、`fire_small_target_motion_min`、`fire_small_target_fire_ratio_min`。
2. 跟踪误关联多：减小 `fire_track_match_dist_px`，增大 `fire_track_min_iou`，减小 `fire_track_area_change_max`。
3. 真实火焰漏检增加：回调上述阈值，优先先下调 `fire_small_target_fire_ratio_min` 和 `fire_small_target_motion_min`。
